# Manual QA Testing – ShadowFox Website

## Project Overview
This project demonstrates end-to-end manual quality assurance testing of a live website. 
It includes test case design, execution, defect identification, and Jira bug tracking 
following Agile SDLC practices.

## Scope of Testing
- Functional testing of key website features
- UI validation and usability checks
- Negative testing and input validation
- Bug reporting and tracking using Jira

## Key Activities
- Designed and executed manual test cases
- Identified functional and UI defects
- Logged defects in Jira with clear reproduction steps
- Followed bug lifecycle (Open → Fixed → Retest → Closed)

## Tools & Methodologies
- Manual Testing
- Jira
- Agile / SDLC
- Browser-based testing (Chrome)

- ## QA Workflow Followed
1. Requirement understanding
2. Test case design
3. Manual test execution
4. Bug identification
5. Bug logging in Jira
6. Retesting and closure


## Outcome
Improved understanding of QA workflows, defect management, and collaboration 
between QA and development teams.
